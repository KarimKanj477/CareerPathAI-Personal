# Software Requirements Specification (SRS)

## 1. Purpose

The purpose of this document is to describe the functional and non-functional requirements of the CareerPath AI application. It serves as a guide for the development team during the implementation of the project.



## 2. Project Scope

CareerPath AI is a web application that helps users choose a career path by analyzing their current skills and comparing them with the skills required for a selected profession.

Based on this analysis, the application generates a personalized learning roadmap and recommends learning resources to help users achieve their career goals.



## 3. User Roles

The system includes the following user roles:

### Student/User

- Create an account
- Log in
- Select a career path
- Add personal skills
- View learning roadmap
- Track progress

### Administrator

- Manage careers
- Manage skills
- Manage learning resources
- Manage users
- View system statistics

---

## 4. Functional Requirements

The system should allow users to:

- Register and log in.
- Manage their personal profile.
- Select a target career.
- Add current skills.
- Analyze skill gaps.
- Generate personalized learning roadmaps.
- Recommend learning resources.
- Track learning progress.
- Receive AI career recommendations.

---

## 5. Non-Functional Requirements

The application should:

- Be easy to use.
- Provide a responsive user interface.
- Protect user information.
- Store data securely.
- Return search results quickly.
- Support modern web browsers.
